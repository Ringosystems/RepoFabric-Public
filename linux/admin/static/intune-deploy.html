Intune policy export

Generates a Microsoft Intune Settings Catalog JSON file that locks managed Windows endpoints onto this WinGet REST source and shuts off the public community + Microsoft Store sources. The file downloads to your machine. Hand it to an Intune admin (or import it yourself via Graph) under their own credentials. back to admin

Export only. This tool does not push to Intune. The minimum write scope Microsoft offers for the configurationPolicies endpoint is DeviceManagementConfiguration.ReadWrite.All, which grants write across every device-config policy in the tenant. We deliberately do not hold that scope inside a single-purpose lockdown tool. The Intune admin imports the JSON file in their own context.

Assignment is out of scope. After import, the Intune admin assigns the policy to a device or group through the Intune portal.

Out of scope for this policy

Per-install silent mode: the DesktopAppInstaller CSP does not have a "force silent" toggle. Silent behaviour is driven by each manifest's InstallerSwitches.Silent field, which this tool already sets when you publish a custom app.

TLS validation for REST sources: winget validates the REST endpoint against the system trust store automatically. There is no CSP toggle to relax or tighten that for sources other than the Microsoft Store. If your server uses an internal CA, that CA must be trusted by the endpoint (Intune trusted-certificate profile).
